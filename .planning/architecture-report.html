The Big Idea

A Self-Improving Team Intelligence Engine

Most knowledge systems are passive — you put things in, you get things out. This system is active: it continuously analyzes what the team stores, reorganizes it into structured clusters, and synthesizes specialized AI agents that get better as more knowledge accumulates. The more the team contributes, the smarter every member's AI becomes.

📥

Capture

Team members store prompt patterns, workflows, domain facts, and anti-patterns via MCP tools.

🧠

Analyze

Background pipeline clusters, deduplicates, scores, and summarizes knowledge using embeddings + LLM.

🤖

Generate Agents

LLM synthesizes specialized agent system prompts from each knowledge cluster. Agents evolve with every pipeline run.

📈

Evolve

Agents are versioned. Each new knowledge contribution can trigger a new, improved agent version with a changelog.

🌐

Browse & Download

Embedded web UI provides full visibility into stored knowledge, clusters, datasets, and agents. Download any agent in any format.

🔌

MCP Protocol

Everything accessible to LLM clients via MCP tools, resources, and prompt templates. No SDK required.

Architecture

System Overview — Single Go Binary

Everything ships in one binary. No separate service deployments. The Go binary runs the MCP server, the web UI server, the analysis pipeline, and the agent generation engine — all within the same process.

Analysis Pipeline

Continuous Knowledge Processing

Runs as background goroutines triggered by new entry count thresholds or a time interval. Each stage feeds the next. The result is a versioned, processed dataset — and a trigger for agent generation.

Agent Generation

Specialized Agents from Knowledge Clusters

After each pipeline run, the engine generates or updates one agent per knowledge cluster. Agents are versioned, carry changelogs, and can be downloaded in multiple formats. As the team contributes more, agents become more specialized and accurate.

🤖 Agent Generation Flow
1Analysis pipeline completes; clusters ready
2For each cluster with ≥ threshold entries: load summary, top-rated entries, anti-patterns
3LLM call: synthesize specialized system prompt
4Diff against previous agent version
5Record changelog: what improved, what was added
6Persist new agent version (draft or published)
7Available via MCP and web UI for download
📦 Agent Export Formats
1Claude Subagent .md — YAML frontmatter + system prompt; drop into .claude/agents/
2System Prompt .txt — Plain text; works with any LLM (GPT, Gemini, etc.)
3JSON — Full structured config: instructions, knowledge refs, anti-patterns, version history
4Bulk ZIP — All agents in chosen format
📊 Agent Properties
·Versioned — v1, v2, v3… with full history preserved
·Changelog — diff summary between versions
·Draft / Published states — curators approve before serving via MCP
·Source knowledge refs — which entries informed this agent
·Cross-domain agents — for clusters spanning multiple domains
·Served directly via MCP resource agents://domain/{name}

Web UI

Embedded Browser Interface

React SPA compiled by Vite and embedded in the Go binary via //go:embed dist/. No separate frontend deployment. Served at http://localhost:{port} when the binary runs.

/ Dashboard
Entry count, cluster count, agent count, pipeline status, recent activity feed
/knowledge
Browse, search, and filter all entries. View detail, rate, edit. Cluster assignment visible.
/clusters
Visual cluster map. Entry count per cluster. LLM-generated cluster summaries.
/datasets
Versioned processed datasets. Before/after pipeline diffs. Download full datasets as JSON or CSV.
/agents
All generated agents. Version history. Status (draft/published). Download individual or bulk ZIP.
/agents/:id
Full agent definition. Source knowledge refs. Diff viewer between versions. Approve/reject draft.
/analytics
Usage heatmaps, top entries, domain coverage gaps, contribution leaderboard.
/settings
Team config, domain taxonomy, pipeline schedule, embedding provider, agent generation thresholds.

Tech Stack

Technology Decisions

Runtime
Go go
Single binary; go:embed for UI; excellent goroutine concurrency for pipeline
MCP Protocol
mark3labs/mcp-go
Most mature Go MCP SDK; handles framing, schema, transport
Storage (v1)
SQLite + sqlite-vec
Zero external deps; embeds anywhere; vector search via extension
Storage (v2)
PostgreSQL + pgvector
Production scale, multi-instance, same query interface
Embeddings
Pluggable
Ollama (local/offline) or OpenAI text-embedding-3-small
LLM (pipeline)
Anthropic API llm
claude-haiku-4-5 for analysis; claude-sonnet-4-6 for agent generation
Frontend
React + Vite react
TypeScript; compiled to static assets; embedded via go:embed
UI Components
shadcn/ui + Tailwind
Rapid, consistent dark-mode UI without custom CSS overhead
Testing
Go testing + Vitest
Native test runners per layer; no cross-language test framework

Boundaries

Out of Scope for v1

Cross-Org Federation

Trust and privacy models for sharing knowledge between organizations require dedicated design work.

SSO / OAuth

Basic team auth for v1. SSO deferred until team auth model is validated.

Fine-Tuning / Training

Neural model training on team data is a future research track once sufficient rated data exists.

Real-time Push

UI uses polling/manual refresh in v1. WebSocket push deferred to v2.

Workflow Automation

Agent runtime orchestration is out of scope. This system generates agents; clients execute them.
