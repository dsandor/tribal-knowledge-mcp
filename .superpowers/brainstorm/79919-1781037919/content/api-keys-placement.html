Where should API Keys management live?

The backend already has full CRUD for keys — this is about where users find it in the UI.

A

Standalone page — /settings/api-keys

Dedicated page linked from the sidebar under Settings. Full table, create dialog, revoke action. Same pattern as Teams and Auth Config pages.

Pros

  • Room to grow (usage stats, expiry)
  • Easy to deep-link from docs

Cons

  • One more nav item
B

Tab inside Settings page

Settings gets a tab bar: General | API Keys. Keys table + create button lives in the API Keys tab.

Pros

  • No new sidebar entry
  • Keeps admin config co-located

Cons

  • Settings page grows; harder to deep-link
C

Inside Admin Teams page

API keys panel added to the existing Teams admin view, since keys are team-scoped.

Pros

  • Mirrors team-centric data model

Cons

  • Conflates two concerns; cramped
  • Hides user-level keys
